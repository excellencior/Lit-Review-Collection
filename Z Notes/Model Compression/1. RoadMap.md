## 3. Phase 1 — Classical Compression Pillars (Weeks 4–8)

> **Goal:** Master the four classical pillars of model compression with both theory and hands-on implementation. These are the building blocks everything else rests on.

### Pillar A: Pruning & Sparsity (Weeks 4–5)

#### Week 4: Unstructured Pruning

| Topic | Detail |
|-------|--------|
| **Magnitude Pruning** | The simplest baseline: remove weights with smallest magnitude. Implement on ResNet-18/CIFAR-10. Plot accuracy vs. sparsity curves (0%, 50%, 70%, 90%, 95%). |
| **Iterative Magnitude Pruning (IMP)** | Prune → retrain → repeat. Much better than one-shot. Implement the full loop. |
| **The Lottery Ticket Hypothesis** | Read the paper carefully. Core claim: dense networks contain sparse subnetworks ("winning tickets") that can train to full accuracy when reset to initial weights. Understand: why this is profound, and why it's expensive. |
| **Rewinding** | Lottery Ticket with Rewinding (Frankle et al., 2019): rewind to early training, not init. More practical. |
| **Sensitivity Analysis** | Different layers have different sensitivity to pruning. Implement per-layer sensitivity analysis: prune each layer independently, measure accuracy drop. |

**Key Papers:**
- [Learning Both Weights and Connections (Han et al., 2015)](https://arxiv.org/abs/1506.02626) — the OG pruning paper
- [Deep Compression (Han et al., 2016)](https://arxiv.org/abs/1510.00149) — pruning + quantization + Huffman coding
- [The Lottery Ticket Hypothesis (Frankle & Carlin, 2019)](https://arxiv.org/abs/1803.03635)
- [Stabilizing the Lottery Ticket Hypothesis (Frankle et al., 2020)](https://arxiv.org/abs/1903.01611)

**Hands-on Deliverable:** A Jupyter notebook that prunes ResNet-18 to 90% sparsity with <2% accuracy loss on CIFAR-10, using IMP with rewinding.

#### Week 5: Structured Pruning & Hardware-Friendly Sparsity

| Topic | Detail |
|-------|--------|
| **Why Unstructured Pruning Doesn't Speed Things Up** | Sparse matrices on GPUs are often _slower_ than dense. Understand why: irregular memory access, lack of hardware support. |
| **Structured Pruning** | Remove entire filters, channels, or attention heads. Results in a genuinely smaller dense model. |
| **N:M Sparsity** | NVIDIA's 2:4 structured sparsity: out of every 4 elements, exactly 2 are zero. Tensor Cores can accelerate this 2×. |
| **Importance Criteria** | Beyond magnitude: Taylor expansion-based, gradient-based, Fisher information-based pruning criteria. |
| **LLM Pruning (Modern)** | **SparseGPT** (2023): one-shot pruning for LLMs using Hessian-based updates. **Wanda** (2023): even simpler — prune by weight × activation magnitude. Both achieve 50% sparsity with minimal loss. |

**Key Papers:**
- [Learning Structured Sparsity (Wen et al., 2016)](https://arxiv.org/abs/1608.03665)
- [Accelerating Sparse Deep Neural Networks (NVIDIA, 2021)](https://arxiv.org/abs/2104.08378) — N:M sparsity
- [SparseGPT (Frantar & Alistarh, 2023)](https://arxiv.org/abs/2301.00774)
- [Wanda (Sun et al., 2023)](https://arxiv.org/abs/2306.11695)

**Hands-on Deliverable:** Implement filter pruning on ResNet-18 (prune 30% of filters per layer). Show real latency speedup (not just FLOP reduction). Then, try running SparseGPT on a small LLM (<3B) using the official code.

### Pillar B: Quantization (Weeks 6–7)

> This is the **most impactful** pillar for practical deployment today. Spend extra time here.

#### Week 6: Quantization Fundamentals & PTQ

| Topic | Detail |
|-------|--------|
| **What is Quantization?** | Mapping continuous/high-precision values to discrete/low-precision values. Understand: scale, zero-point, symmetric vs. asymmetric, per-tensor vs. per-channel vs. per-group. |
| **Uniform vs. Non-Uniform** | Uniform: evenly spaced grid (INT8). Non-Uniform: k-means clustering, codebook-based (lookup table quantization). Understand tradeoffs. |
| **Calibration** | For PTQ, you need calibration data to determine scale/zero-point. Methods: min-max, percentile, MSE minimization, entropy (KL divergence). |
| **Post-Training Quantization (PTQ)** | Quantize after training. No retraining needed. Fast and cheap. But quality degrades at low bit-widths. |
| **INT8 PTQ (the bread and butter)** | Implement on ResNet-18 using PyTorch's quantization API. Compare Dynamic Quantization vs. Static Quantization. Measure accuracy, latency, model size. |
| **The Activation Outlier Problem** | Some transformer layers produce massive activation outliers (e.g., 100× larger than the rest). These destroy INT8 quantization. This is the central challenge of LLM quantization. |

**Key Papers:**
- [Quantization and Training of Neural Networks for Efficient Integer-Arithmetic-Only Inference (Jacob et al., 2018)](https://arxiv.org/abs/1712.05877) — the Google quantization paper
- [A Survey of Quantization Methods for Efficient Neural Network Inference (Gholami et al., 2021)](https://arxiv.org/abs/2106.08295)

**Hands-on Deliverable:** PTQ INT8 on ResNet-18, with a full report: accuracy drop, latency speedup, model size reduction. Try both torch static and ONNX Runtime quantization.

#### Week 7: Quantization-Aware Training & Advanced Methods

| Topic | Detail |
|-------|--------|
| **Quantization-Aware Training (QAT)** | Insert fake-quant ops during training. The model learns to be robust to quantization noise. Better than PTQ, but expensive. Understand the Straight-Through Estimator (STE). |
| **Mixed-Precision Quantization** | Not all layers need the same bit-width. Sensitive layers get more bits, others get fewer. Methods: HAQ (hardware-aware), HAWQ (Hessian-based). |
| **Second-Order Methods for Quantization** | **OBQ (Optimal Brain Quantizer):** uses Hessian information to compensate for quantization error. Foundation for GPTQ. Understand the math: row-wise Hessian inverse, greedy quantization with weight update. |
| **Binary/Ternary Networks** | Extreme quantization: 1-bit weights (±1) or 2-bit (−1, 0, +1). Massive compression but significant accuracy loss. Understand XNOR-Net, BNN, and why this remains hard. |
| **QAT Implementation** | Train a CNN on CIFAR-10 with QAT using PyTorch FX. Compare with PTQ at INT8 and INT4. |

**Key Papers:**
- [Quantizing Deep CNNs for Efficient Inference (Krishnamoorthi, 2018)](https://arxiv.org/abs/1806.08342) — Google's QAT whitepaper
- [HAQ: Hardware-Aware Automated Quantization (Wang et al., 2019)](https://arxiv.org/abs/1811.08886)
- [HAWQ-V3 (Yao et al., 2021)](https://arxiv.org/abs/2011.10680)
- [Optimal Brain Compression (Frantar & Alistarh, 2022)](https://arxiv.org/abs/2208.11580) — the OBQ paper

### Pillar C: Knowledge Distillation (Week 8, Part 1)

| Topic | Detail |
|-------|--------|
| **Core Idea** | A small "student" model learns from a large "teacher" model's soft predictions, not just hard labels. The teacher's probability distribution over all classes carries "dark knowledge." |
| **Temperature Scaling** | Softmax with temperature T>1 produces softer distributions. Higher T → more informative (but noisier) supervision. Find the sweet spot. |
| **Loss = α·KD_Loss + (1−α)·CE_Loss** | Interpolate between distillation loss (teacher's soft targets) and the standard cross-entropy with hard labels. Tune α and T. |
| **Feature/Intermediate Distillation** | Don't just match outputs — match intermediate representations. FitNets, Attention Transfer, CRD. More powerful than logit-only KD. |
| **Self-Distillation** | The model teaches itself: train → distill → retrain. Born-Again Networks. Surprisingly effective. |
| **KD for Transformers** | **DistilBERT** (6-layer BERT → 40% smaller, 60% faster, 97% of accuracy). **TinyBERT** (data augmentation + intermediate distillation). **MiniLM** (self-attention distillation). |

**Key Papers:**
- [Distilling the Knowledge in a Neural Network (Hinton et al., 2015)](https://arxiv.org/abs/1503.02531)
- [FitNets (Romero et al., 2015)](https://arxiv.org/abs/1412.6550)
- [DistilBERT (Sanh et al., 2019)](https://arxiv.org/abs/1910.01108)
- [TinyBERT (Jiao et al., 2020)](https://arxiv.org/abs/1909.10351)
- [MiniLM (Wang et al., 2020)](https://arxiv.org/abs/2002.10957)

**Hands-on Deliverable:** Distill ResNet-50 → ResNet-18 on CIFAR-10 using logit KD. Then add intermediate feature matching. Compare both against training ResNet-18 from scratch.

## 4. Phase 2 — LLM-Era Compression (Weeks 9–14)

> **Goal:** This is where the field is TODAY. Everything modern revolves around compressing LLMs (7B–70B+ parameters). This is likely where your thesis lives.

### LLM Quantization: The Main Event (Weeks 9–11)

#### Week 9: The Outlier Problem & First Solutions

| Topic | Detail |
|-------|--------|
| **Why LLMs are hard to quantize** | Activation outliers: certain channels produce values 100× larger than others. Naive INT8 quant destroys accuracy because the scale factor is dominated by outliers. |
| **LLM.int8()** | Dettmers et al. (2022). Key insight: decompose into "outlier" channels (kept in FP16) and "normal" channels (INT8). Mixed-precision decomposition at inference time. ~0% accuracy loss at INT8. |
| **SmoothQuant** | Xiao et al. (2023). Key insight: mathematically migrate the quantization difficulty from activations to weights using a smoothing factor `s`. Per-channel division on activations, multiplication on weights. Enables W8A8 quantization. |
| **Understand the math** | Write out the linear layer: `Y = XW`. SmoothQuant transforms this to `Y = (X · diag(s)^-1) · (diag(s) · W)`. Derive why this helps: activation ranges shrink, weight ranges grow (weights are easier to quantize). |

**Key Papers:**
- [LLM.int8() (Dettmers et al., 2022)](https://arxiv.org/abs/2208.07339)
- [SmoothQuant (Xiao et al., 2023)](https://arxiv.org/abs/2211.10438)

**Hands-on:** Run LLM.int8() on a 7B model using `bitsandbytes`. Measure perplexity on WikiText-2. Compare with FP16 baseline.

#### Week 10: Weight-Only Quantization (4-bit and Below)

| Topic | Detail |
|-------|--------|
| **Why weight-only?** | LLM inference is memory-bandwidth-bound (not compute-bound). Quantizing weights reduces memory → faster token generation. Activations stay in FP16. |
| **GPTQ** | Frantar et al. (2023). Row-by-row quantization using approximate second-order information (Hessian inverse). Based on Optimal Brain Quantizer. Enables accurate 4-bit and 3-bit weight quantization for LLMs. **Study this algorithm carefully** — it's a foundation for many follow-ups. |
| **AWQ (Activation-Aware Weight Quantization)** | Lin et al. (2024). Key insight: not all weights are equally important. Weights corresponding to high-activation channels are more salient. Scale salient channels up before quantization to reduce their relative error. Simpler than GPTQ, often comparable quality. |
| **QuIP / QuIP#** | Chee et al. (2023, 2024). Use random orthogonal rotations (incoherence processing) to make weight matrices more quantization-friendly. Enables accurate 2-bit quantization. |
| **AQLM** | Egiazarian et al. (2024). Additive Quantization for LLMs: use multi-codebook representations. State-of-the-art at 2-bit. |

**Key Papers:**
- [GPTQ (Frantar et al., 2023)](https://arxiv.org/abs/2210.17323)
- [AWQ (Lin et al., 2024)](https://arxiv.org/abs/2306.00978)
- [QuIP (Chee et al., 2023)](https://arxiv.org/abs/2307.13304)
- [QuIP# (Tseng et al., 2024)](https://arxiv.org/abs/2402.04396)
- [AQLM (Egiazarian et al., 2024)](https://arxiv.org/abs/2401.06118)

**Hands-on:** Quantize Llama-2-7B or Llama-3-8B with GPTQ (4-bit) and AWQ (4-bit). Compare: perplexity, generation speed, memory usage. Use AutoGPTQ and the AWQ library.

#### Week 11: Rotation-Based Quantization & QLoRA

| Topic | Detail |
|-------|--------|
| **QuaRot** | Ashkboos et al. (2024). Apply Hadamard rotations to both weights and activations to eliminate outliers. Enables outlier-free 4-bit inference (W4A4). No calibration data needed for rotations. |
| **SpinQuant** | Liu et al. (2024, Meta). Learn the optimal rotation matrices (instead of random/Hadamard). Better than QuaRot. |
| **QLoRA** | Dettmers et al. (2023). 4-bit NormalFloat quantization + LoRA adapters for fine-tuning. Enables fine-tuning a 65B model on a single 48GB GPU. Understand: nf4 data type, double quantization, paged optimizers. |
| **The PEFT Connection** | LoRA (Low-Rank Adaptation) itself: freeze base weights, add low-rank A·B matrices. Understand: rank selection, which layers to adapt, merge strategies. |

**Key Papers:**
- [QuaRot (Ashkboos et al., 2024)](https://arxiv.org/abs/2404.00456)
- [SpinQuant (Liu et al., 2024)](https://arxiv.org/abs/2405.16406)
- [QLoRA (Dettmers et al., 2023)](https://arxiv.org/abs/2305.14314)
- [LoRA (Hu et al., 2021)](https://arxiv.org/abs/2106.09685)

**Hands-on:** Fine-tune Llama-3-8B on a domain task using QLoRA (4-bit). Compare quality with full fine-tuning baseline (on a smaller model if GPU-constrained).

### KV-Cache Quantization (Week 12)

> The KV-cache is the **silent memory killer** for long-context LLM serving. This is a hot research area.

| Topic | Detail |
|-------|--------|
| **The Problem** | KV-cache grows linearly with sequence length: `Memory = 2 × L × H × D × S × bytes_per_element`. For Llama-2-70B at 4K context with FP16: ~2.5 GB per request. At 128K context: ~80 GB. This is the bottleneck for batch size and context length. |
| **KVQuant** | Hooper et al. (NeurIPS 2024). Per-channel key quantization, per-token value quantization. Nuanced treatment: keys and values have different distribution properties. Non-uniform (NUQ) codebooks for values. Achieves <0.1 perplexity degradation at 3-bit. |
| **KIVI** | Liu et al. (2024). 2-bit asymmetric quantization. Key insight: quantize keys per-channel, values per-token. Residual correction for early tokens (positional bias). |
| **Why Keys ≠ Values** | Keys tend to have per-channel outlier patterns (consistent across tokens). Values tend to have per-token outlier patterns (vary by position). Different quantization strategies needed. |

**Key Papers:**
- [KVQuant (Hooper et al., 2024)](https://arxiv.org/abs/2401.18079)
- [KIVI (Liu et al., 2024)](https://arxiv.org/abs/2402.02750)
- [Gear: An Efficient KV Cache Compression Recipe (Kang et al., 2024)](https://arxiv.org/abs/2403.05527)

**Hands-on:** Implement KV-cache quantization in a simple GPT inference loop. Measure: memory savings, perplexity impact at 4-bit, 3-bit, 2-bit.

### Knowledge Distillation for LLMs (Weeks 13–14)

#### Week 13: LLM Distillation Fundamentals

| Topic | Detail |
|-------|--------|
| **Challenges unique to LLMs** | Teacher is autoregressive → output space is combinatorial. Standard KD (logit matching) at every token is expensive (full teacher forward pass at every step). |
| **Sequence-Level KD** | Kim & Rush (2016). Generate translations/outputs from teacher, then train student on them as if they were gold data. Simple and effective. |
| **On-Policy Distillation (GKD)** | Agarwal et al. (2024, Google). Student generates its own outputs, teacher scores them. Avoids train-time distribution mismatch. State-of-the-art for LLM KD. |
| **MiniLLM** | Gu et al. (2023). Minimize reverse KL divergence (student → teacher) instead of forward KL. Prevents student from spreading probability mass too thin over modes it can't model. |

**Key Papers:**
- [Sequence-Level Knowledge Distillation (Kim & Rush, 2016)](https://arxiv.org/abs/1606.07947)
- [GKD: Generalized Knowledge Distillation (Agarwal et al., 2024)](https://arxiv.org/abs/2306.13649)
- [MiniLLM (Gu et al., 2023)](https://arxiv.org/abs/2306.08543)

#### Week 14: Distillation in Practice + Hybrid Compression

| Topic | Detail |
|-------|--------|
| **Distillation Recipes in the Wild** | How were Phi-1/2/3, Gemma, MiniCPM, OpenELM actually trained? Many use synthetic data from larger models (a form of KD). Study the training recipes. |
| **TinyLLM** | Multi-teacher KD for small LLMs. You have this paper in your collection — read it carefully. |
| **Hybrid: KD + Quantization** | Train a distilled student → then quantize it → compare Pareto front against directly quantizing the large teacher. This is a strong thesis direction. |
| **Hybrid: KD + Pruning** | Prune the teacher → use pruned teacher as student init → distill. Or prune the student after distillation. Order matters — experiment. |

**Hands-on Deliverable:** Distill a 7B model to a 1.3B student using sequence-level KD. Then quantize the student to 4-bit. Plot: (accuracy, latency, memory) Pareto front vs. directly quantizing the 7B teacher.

## 5. Phase 3 — Systems, Serving & Hardware Co-Design (Weeks 15–18)

> **Goal:** Compression without deployment is academic exercise. Learn the systems stack that turns compressed models into real-world speed gains.

### Week 15: Efficient Attention Mechanisms

| Topic | Detail |
|-------|--------|
| **FlashAttention** | Dao et al. (2022, 2023). IO-aware attention: fuse attention computation to avoid materializing the N×N attention matrix in HBM. Understand: tiling, online softmax, memory hierarchy exploitation. |
| **FlashAttention-2** | Better parallelism, 2× faster. Understand the key improvements. |
| **Multi-Query Attention (MQA)** | Share K, V heads across all Q heads. 8× KV-cache reduction. Used in PaLM, Falcon. |
| **Grouped-Query Attention (GQA)** | Interpolation: group Q heads to share fewer K, V heads. Used in Llama-2-70B, Llama-3. Sweet spot between MQA and MHA. |
| **Linear Attention & Alternatives** | Performers, Random Feature Attention, Mamba/S4 (state-space models). Subquadratic attention. Understand the tradeoffs vs. softmax attention. |

**Key Papers:**
- [FlashAttention (Dao et al., 2022)](https://arxiv.org/abs/2205.14135)
- [FlashAttention-2 (Dao, 2023)](https://arxiv.org/abs/2307.08691)
- [GQA (Ainslie et al., 2023)](https://arxiv.org/abs/2305.13245)
- [Mamba (Gu & Dao, 2023)](https://arxiv.org/abs/2312.00752)

### Week 16: Inference Serving Systems

| Topic | Detail |
|-------|--------|
| **vLLM & PagedAttention** | Kwon et al. (2023). Paged memory management for KV-cache (like virtual memory). Eliminates memory fragmentation. 2–4× throughput improvement. Understand: logical/physical KV blocks, copy-on-write. |
| **Continuous Batching** | Orca (2022). Don't wait for the longest sequence in a batch to finish. Dynamically insert/remove sequences from the batch. 10×+ throughput over static batching. |
| **Speculative Decoding** | Leviathan et al. (2023). Use a small draft model to propose multiple tokens, verify with the large model in parallel. No quality loss (exact same distribution). 2–3× speedup. |
| **TensorRT-LLM** | NVIDIA's optimized inference engine. FP8/INT8/INT4 kernel fusion, KV-cache quantization, in-flight batching. Learn to deploy and benchmark. |

**Key Papers:**
- [vLLM / PagedAttention (Kwon et al., 2023)](https://arxiv.org/abs/2309.06180)
- [Speculative Decoding (Leviathan et al., 2023)](https://arxiv.org/abs/2211.17192)
- [Orca (Yu et al., 2022)](https://www.usenix.org/conference/osdi22/presentation/yu)

**Hands-on:** Deploy a 4-bit quantized model on vLLM. Benchmark: TTFT (time to first token), throughput (tok/s), latency (p50/p95). Then deploy the same model on TensorRT-LLM with FP8 and compare.

| Topic | Detail |

## 11. Thesis Directions (Pick One, Go Deep)

### Direction 1: LLM PTQ Beyond 4-bit + Robust KV-Cache Quantization
- **Goal:** Achieve 3-/2-bit weight quantization with minimal perplexity loss, coupled with quantized KV-cache for long-context serving.
- **Angle:** Rotation/outlier-aware grouping + data-free calibration; fuse with KVQuant/KIVI-style cache quantization.
- **Why now:** Memory is the serving bottleneck; FP4/FP8 hardware is arriving. Strong publishability at venues like NeurIPS, ICML.
- **Key baseline papers:** GPTQ, AWQ, QuaRot, SpinQuant, KVQuant, KIVI

### Direction 2: Hybrid KD + Quantization Students for Domain Tasks
- **Goal:** Train a student ≤1–3B via KD, then 4-bit PTQ/QAT; beat teacher-PTQ on quality/latency per $/W.
- **Angle:** Attention-map KD + layerwise loss scheduling; then GPTQ/QAT with activation range learning.
- **Why now:** Small language models (Phi, Gemma, MiniCPM) are a hot trend. Combining KD with aggressive quantization is underexplored.
- **Key baseline papers:** GKD, MiniLLM, GPTQ, QLoRA, TinyLLM (in your collection)

### Direction 3: Structured Sparsity + Low-Bit Co-Design
- **Goal:** 2:4 (or similar) sparsity + 8/4-bit on Tensor Cores; prove real speedups at batch≥1.
- **Angle:** Pruning during finetune + sparse kernels; report end-to-end gains, not just FLOPs.
- **Why now:** N:M sparsity has hardware support but few papers show real end-to-end gains combined with quantization.
- **Key baseline papers:** SparseGPT, Wanda, NVIDIA N:M sparsity, GPTQ

### Direction 4: Compression-Aware Serving
- **Goal:** Co-design compression with inference serving optimizations (speculative decoding, paged attention) for cluster-level throughput wins.
- **Angle:** Speculative decoding with a compressed draft model; quantized KV-cache + paged attention; continuous batching-aware compression.
- **Why now:** Inference cost dominates LLM deployments. Systems-level optimization is a rich research area.
- **Key baseline papers:** vLLM, Speculative Decoding, TensorRT-LLM, StreamingLLM

### Direction 5: Fairness-Aware Model Compression
- **Goal:** Develop compression techniques that maintain fairness guarantees across demographic groups.
- **Angle:** Constrained pruning/quantization that minimizes accuracy disparity; fairness-regularized distillation.
- **Why now:** Entirely underexplored. You already have 4+ papers on this topic in your collection. High novelty.
- **Key baseline papers:** Your existing collection papers + standard compression baselines

## 12. Evaluation Recipe (What Reviewers Expect)

### Baselines (always include)
- FP16/BF16 full-precision (upper bound)
- FP8 (the new default baseline)
- INT8 PTQ (the classic baseline)
- GPTQ-4b, AWQ-4b (weight-only 4-bit baselines)
- QLoRA-4b (if fine-tuning)

### Quality Metrics
| Task Type | Metrics |
|-----------|---------|
| **LLMs** | Perplexity (WikiText-2, C4, PG19), zero-shot accuracy (ARC, HellaSwag, WinoGrande, PIQA, MMLU) |
| **Encoders (BERT etc.)** | GLUE benchmark, SQuAD F1 |
| **Vision** | ImageNet top-1 accuracy, MS-COCO mAP |

### Efficiency Metrics
- **Latency:** Time per token (ms), time to first token (TTFT), p50/p95
- **Throughput:** Tokens/sec (generation), images/sec (vision)
- **Memory:** Peak VRAM (GB), model file size, KV-cache size at context length N
- **Params:** Total, active (for MoE), after pruning
- **Energy:** GPU power draw (nvidia-smi), Joules per token/image

### Ablations Reviewers Love
- Calibration set size vs. quality
- Bit-mixing experiments (W4A8, W3A8, W4A4, W2A16)
- KV-cache bitwidth vs. throughput at increasing context lengths
- Per-layer sensitivity analysis
- Distillation loss type comparison (logit vs. feature vs. attention)
- Pruning ratio vs. quantization bitwidth interaction

## 13. Future Prospects (Where the Field is Headed)

### Near-Term (2025–2026)
- **FP4 becomes mainstream:** NVIDIA Blackwell-class GPUs with native FP4 Tensor Cores. Design experiments to be FP4-ready.
- **W4A4 inference:** Full 4-bit (weights AND activations). QuaRot/SpinQuant-like rotation methods enabling this.
- **KV-cache compression standardized:** 4-bit or 3-bit KV-cache becoming default in serving frameworks.
- **Speculative decoding + compressed drafters:** Small quantized draft models for speculative decoding.

### Medium-Term (2026–2028)
- **1-bit and sub-1-bit models:** BitNet-style architectures trained from scratch with ternary weights. If these scale, everything changes.
- **Compression-aware pre-training:** Models pre-trained knowing they will be compressed. Architecture choices baked in.
- **MoE compression:** As MoE models (Mixtral, DeepSeek) become standard, compressing their experts becomes critical and underexplored.
- **Hybrid compression stacks:** KD + pruning + quantization + NAS co-designed with hardware kernels and compilers. End-to-end differentiable compression.

### Long-Term (2028+)
- **Analog/photonic compute:** Fundamentally different compute substrates where "quantization" means something entirely different.
- **Model-hardware co-evolution:** Chips designed around specific model structures; models designed for specific chips.
- **Biological efficiency inspiration:** The brain runs on ~20W. What can we learn?
